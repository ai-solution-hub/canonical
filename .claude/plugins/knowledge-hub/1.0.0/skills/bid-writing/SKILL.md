# Bid Writing

Expert guidance for UK public procurement bid responses. This skill covers
terminology, response structure, confidence assessment, evidence presentation,
word limit management, and quality standards specific to UK tender submissions.

## UK Public Procurement Context

### Terminology

| Term                 | Meaning                                                                 |
| -------------------- | ----------------------------------------------------------------------- |
| **PQQ**              | Pre-Qualification Questionnaire — initial screening stage               |
| **SQ**               | Selection Questionnaire (newer name for PQQ)                            |
| **ITT**              | Invitation to Tender — full bid submission                              |
| **ITQ**              | Invitation to Quote — simpler procurement                               |
| **Tender**           | The formal request for proposals/bids                                   |
| **Lot**              | A subset of a framework or contract                                     |
| **Framework**        | A pre-approved list of suppliers for call-off contracts                 |
| **Call-off**         | An order placed under an existing framework                             |
| **MEAT**             | Most Economically Advantageous Tender — evaluation criteria             |
| **MAT**              | Most Advantageous Tender — replaces MEAT under the Procurement Act 2023 |
| **Social value**     | Community benefit — often 10-20% of evaluation weighting                |
| **Method statement** | Detailed description of how you will deliver                            |
| **Case study**       | Evidence of previous delivery — real examples                           |
| **Scoring matrix**   | How the buyer will score responses (pass/fail, 0-5, etc.)               |

### Procurement Stages

```
PQQ/SQ                    ITT                     Contract Award
(Pass/Fail screening) --> (Scored submission) --> (Preferred bidder)
                                                         |
- Organisation details    - Method statements            v
- Financial standing      - Case studies              Standstill period
- Insurance levels        - Pricing schedule          (10 calendar days)
- Capability evidence     - Social value                   |
- References              - Implementation plan            v
                          - Risk management            Contract start
```

### Scoring Systems

Most UK public sector bids use a 0-5 scoring system:

| Score | Meaning      | What the evaluator is looking for                             |
| ----- | ------------ | ------------------------------------------------------------- |
| 0     | Unacceptable | No response, or completely fails to address the question      |
| 1     | Poor         | Significant gaps, vague, generic — could be any company       |
| 2     | Acceptable   | Addresses the question but lacks evidence or specificity      |
| 3     | Good         | Solid response with some evidence — meets requirements        |
| 4     | Very Good    | Detailed, evidence-based response exceeding requirements      |
| 5     | Excellent    | Outstanding — specific, evidence-rich, innovative, measurable |

**The difference between a 3 and a 5 is specificity.** Generic descriptions of
processes score 3. Specific examples with named standards, real metrics, and
concrete evidence score 5.

## Response Structure

### Question Type Analysis

Before drafting, classify the question:

| Question Type    | Key Phrase                       | Response Approach                              |
| ---------------- | -------------------------------- | ---------------------------------------------- |
| **Descriptive**  | "Describe your approach to..."   | Lead with methodology, support with evidence   |
| **Evidential**   | "Provide evidence of..."         | Lead with examples, support with process       |
| **Process**      | "How do you ensure/manage..."    | Lead with process steps, support with outcomes |
| **Capability**   | "Demonstrate your ability to..." | Lead with credentials, support with examples   |
| **Compliance**   | "Confirm that you meet..."       | Lead with confirmation, support with detail    |
| **Scenario**     | "How would you handle..."        | Lead with approach, show you've done it before |
| **Social value** | "How will you deliver..."        | Lead with commitment, quantify impact          |
| **Risk**         | "Identify risks and..."          | Lead with risk framework, show mitigation      |

### Standard Response Framework

```
1. OPENING (10-15% of word count)
   - Directly answer the question in 1-2 sentences
   - Signal what the rest of the response will cover
   - Never start with "We" — vary your openings

2. BODY (60-70% of word count)
   - Structured evidence and explanation
   - Use subheadings for responses >300 words
   - Each paragraph should make one clear point
   - Back every claim with evidence (standards, metrics, examples)

3. EVIDENCE (15-20% of word count)
   - Case studies or examples (minimum 1 per response)
   - Specific metrics and outcomes
   - Named standards and certifications

4. CLOSING (5-10% of word count)
   - Measurable outcome or commitment
   - Forward-looking statement tied to the buyer's needs
   - Never end with generic "we look forward to..." phrasing
```

### Opening Sentence Patterns

**Strong openings (use these):**

- "[Organisation] maintains a comprehensive approach to [topic], underpinned by
  [standard/certification]."
- "Our [topic] methodology has been refined over [N] years of delivering
  [service type], achieving [specific outcome]."
- "With [certification] accreditation and [N] successful [project type]
  deliveries, we bring proven capability in [area]."
- "Central to our approach is [key principle], demonstrated through [evidence]."

**Weak openings (avoid these):**

- "We have a policy for..." (too generic)
- "We are committed to..." (tells, doesn't show)
- "We understand the importance of..." (filler)
- "In today's fast-paced environment..." (cliche)

## Confidence Postures

When drafting responses, assess the confidence level based on available KB
material:

### strong_match

**Criteria:** Multiple fresh, high-quality KB sources directly address the
question. Q&A pairs, policies, and case studies all available.

**Approach:**

- Write with authority and specificity
- Include multiple evidence points
- Use precise metrics and dates
- Reference specific standards and certifications
- Include case study detail with measurable outcomes
- Target 95% of word limit

### partial_match

**Criteria:** Some relevant KB sources exist but gaps remain. May have Q&A pairs
but no case studies, or policies but no metrics.

**Approach:**

- Write confidently about what the KB covers
- Clearly mark areas needing additional input with `[SME INPUT NEEDED: topic]`
- Structure the response so gaps can be filled without restructuring
- Use framework language where specific detail is missing
- Target 85-90% of word limit (leaving room for additions)

### needs_sme

**Criteria:** KB has limited coverage. Topic is referenced but not substantively
covered. Need subject matter expert input for a quality response.

**Approach:**

- Draft a structural framework with the right sections
- Fill in what the KB provides
- Mark all gaps clearly: `[SME: please provide specific detail on ...]`
- Include prompting questions to help the SME fill gaps efficiently
- Target 70% of word limit (SME will expand)

### no_content

**Criteria:** No relevant KB material found. This is a knowledge gap.

**Approach:**

- Create a placeholder response structure
- Mark the entire response as needing input
- Suggest which team members or sources might have the answer
- Recommend creating a KB item for this topic after the bid
- Note the gap for future content creation priorities

## Word Limit Management

### Target Usage

| Limit          | Target Range | Rationale                                  |
| -------------- | ------------ | ------------------------------------------ |
| <200 words     | 90-100%      | Short answers must be precise and complete |
| 200-500 words  | 90-95%       | Good responses use the space available     |
| 500-1000 words | 85-95%       | Longer responses can be slightly under     |
| >1000 words    | 80-95%       | Very long limits allow more flexibility    |

Under-utilising the word limit is a common bid writing failure. Evaluators
expect you to use the space provided — a 300-word response to a 500-word
question signals insufficient depth.

### Word Count Strategy

1. **Outline first** — allocate words to each section before writing
2. **Draft at 100%** — write the full response without worrying about limits
3. **Edit to target** — trim or expand to hit the target range
4. **Preserve evidence** — when trimming, cut generic descriptions before
   cutting specific evidence

### If Over Limit

Remove in this order:

1. Redundant phrases ("As mentioned above", "It is worth noting that")
2. Generic context-setting that doesn't score marks
3. Secondary examples (keep the strongest)
4. Descriptive adjectives that don't add meaning

### If Under Limit

Add in this order:

1. A second case study or example
2. More specific metrics and outcomes
3. Named standards or certifications
4. Forward-looking commitment with measurables

## Evidence Presentation

### Case Study Format

For evidential questions, present case studies consistently:

```
**[Client/Sector] — [Project Type] ([Year])**
- **Context**: [1-2 sentences describing the client need]
- **Approach**: [2-3 sentences on what you did]
- **Outcome**: [Specific, measurable results]
  - [Metric 1]: [Number] [description]
  - [Metric 2]: [Number] [description]
```

**Rules:**

- Anonymise client names if required: "a major NHS Trust" not "NHS Trust X"
- Always include at least one measurable outcome
- Recency matters: prefer examples from the last 3 years
- Relevance matters more than impressiveness: a small project in the right
  sector beats a large project in the wrong one
- Check freshness of KB sources: stale case studies may reference outdated facts

### Standards and Certifications

When referencing standards:

- Name the specific standard: "ISO 27001:2022" not just "ISO certified"
- State the scope: "covering all information processing facilities" not just "we
  hold certification"
- Reference audit cycle: "subject to annual surveillance audits by [body]"
- Note any relevant annexes or controls specifically relevant to the question

### Metrics and Evidence Types

| Evidence Type             | Strength  | Example                                                |
| ------------------------- | --------- | ------------------------------------------------------ |
| **Certified outcomes**    | Strongest | "Achieved 99.97% uptime (independently audited)"       |
| **Named metrics**         | Strong    | "Reduced processing time from 14 days to 3 days"       |
| **Comparative data**      | Good      | "Delivering 40% above the SLA target"                  |
| **Process evidence**      | Moderate  | "Reviews are conducted quarterly by senior management" |
| **Commitment statements** | Weak      | "We will ensure high quality delivery"                 |

**Rule:** Every response should contain at least one piece of evidence from the
top three tiers. Commitment statements alone never score above a 3.

## UK English Conventions

All bid responses must use UK English consistently:

### Spelling

| Use                               | Not                                |
| --------------------------------- | ---------------------------------- |
| organisation                      | organization                       |
| colour                            | color                              |
| centre                            | center                             |
| programme (for planned work)      | program (except software programs) |
| licence (noun) / license (verb)   | license for both                   |
| practise (verb) / practice (noun) | practice for both                  |
| analyse                           | analyze                            |
| catalogue                         | catalog                            |
| defence                           | defense                            |
| favour                            | favor                              |
| honour                            | honor                              |
| labour                            | labor                              |
| neighbour                         | neighbor                           |
| realise                           | realize                            |
| specialise                        | specialize                         |
| standardise                       | standardize                        |

### Date Format

- **Always DD/MM/YYYY**: 15/03/2026 not 03/15/2026 or March 15, 2026
- In narrative: "15 March 2026" not "March 15, 2026"
- Relative dates: "Q2 2025/26" for financial year references

### Numbers

- Spell out one to nine, use numerals for 10 and above
- Always use numerals for: percentages (5%), measurements, financial figures
- Use commas as thousands separator: 1,000 not 1000
- Currency: GBP or pounds (not dollars)

### Tone

- Professional and authoritative, not casual
- Active voice preferred: "We deliver" not "Delivery is undertaken"
- Confident but not arrogant: evidence-based assertions, not bold claims
- Client-focused: "Your organisation will benefit from" not "Our service
  provides"

## Quality Indicators

### Signs of a Strong Response (Score 4-5)

- Specific metrics with context ("reduced from X to Y, a Z% improvement")
- Named standards and certifications with scope detail
- Real case studies with measurable outcomes
- Clear structure with logical flow
- Answers the specific question asked (not a generic capability statement)
- Forward-looking commitments tied to the buyer's stated needs
- Uses the buyer's terminology from the tender documents
- Word limit used effectively (90-95%)

### Signs of a Weak Response (Score 1-2)

- Generic descriptions that could apply to any company
- Vague commitments without evidence ("We always strive for excellence")
- No case studies or examples
- Does not directly answer the question asked
- Jargon or buzzwords without substance
- Significantly under the word limit
- Passive voice throughout
- No measurable outcomes referenced

### Common Anti-Patterns

| Anti-Pattern                         | Why it fails                         | Instead                                                                                       |
| ------------------------------------ | ------------------------------------ | --------------------------------------------------------------------------------------------- |
| "We have extensive experience in..." | Generic, unverifiable                | "Over 8 years delivering [specific service] to [sector] clients"                              |
| "We are fully committed to..."       | Commitment is not evidence           | "[Certification] demonstrates our commitment through [specific practice]"                     |
| "Our team of experts..."             | Unsubstantiated claim                | "[N] team members hold [specific qualification], with an average of [N] years' experience"    |
| "We will work closely with..."       | Process description without evidence | "Our collaborative approach, as demonstrated in [case study], involves [specific activities]" |
| Copying the question back            | Wastes word count                    | Start with your answer, not the question                                                      |
| Listing capabilities without context | Tells, doesn't show                  | For each capability, provide one specific example of it in action                             |

## Section-Specific Guidance

### Method Statements

Method statements are the "how will you do it" responses. Structure as:

```
1. Approach overview (2-3 sentences)
2. Key stages/phases with brief description
3. Specific tools, methodologies, or standards applied
4. Quality assurance measures
5. Risk mitigation within the methodology
6. Evidence of methodology working (case study)
7. How you will tailor this for the buyer's specific context
```

### Social Value Responses

Social value is often 10-20% of evaluation. Structure as:

```
1. Your social value commitment specific to THIS contract
2. Measurable targets (not aspirations)
3. How you will evidence and report delivery
4. Previous social value achievements with metrics
5. Local impact (if geographically relevant)
```

Use specific commitments: "We will create [N] apprenticeship positions" not "We
will support apprenticeships."

### Implementation/Mobilisation Plans

Structure as a timeline:

```
Week 1-2: [Activity] — [Deliverable]
Week 3-4: [Activity] — [Deliverable]
Month 2:  [Activity] — [Deliverable]
Month 3:  [Activity] — [Deliverable]
[Ongoing]: [Activity] — [Deliverable]
```

Include: key milestones, dependencies, risk points, and how you will communicate
progress.

## Response Template

Use this structure as a starting point for drafting:

```
## [Question Reference]

[Opening statement — 1-2 sentences directly answering the question]

### [Subheading 1: Core Approach]

[2-3 paragraphs covering the main methodology/approach]
[Reference specific standards, frameworks, or certifications]

### [Subheading 2: Evidence]

**[Case Study: Client/Sector — Project Type (Year)]**
- Context: [What the client needed]
- Approach: [What you did]
- Outcome: [Measurable results]

### [Subheading 3: Your Commitment]

[Forward-looking commitment specific to this contract]
[Measurable targets where possible]

---
Word count: [N] / [Limit]
Confidence: [strong_match | partial_match | needs_sme | no_content]
Sources: [KB item references]
```
